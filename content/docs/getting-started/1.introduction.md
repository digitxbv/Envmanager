# Introduction to EnvManager

EnvManager is a secure, collaborative platform for managing environment variables across your team and projects.

## Why EnvManager?

Managing environment variables is a common challenge for development teams:

- **Security risks** - Sharing secrets via Slack, email, or `.env` files in repos
- **Sync issues** - Team members working with outdated configurations
- **No audit trail** - No visibility into who changed what and when
- **Access control** - Difficulty managing who can see sensitive values

EnvManager solves these problems by providing:

- **Encrypted storage** - All secrets are encrypted at rest using industry-standard encryption
- **Team collaboration** - Share variables securely with role-based access control
- **Environment separation** - Manage dev, staging, and production configs separately
- **Audit logging** - Track every change with timestamps and user attribution

## Key Features

### Multi-tenant Projects
Create separate projects for each application or service, keeping configurations organized and isolated.

### Environment Separation
Each project supports multiple environments (development, staging, production), making it easy to manage different configurations.

### Secure Secrets
Mark sensitive values as secrets. These are encrypted and can only be revealed by authorized team members.

### Team Collaboration
Invite team members with different access levels. Admins can manage settings while members can view and use variables.

### Audit Logging
Every change is logged with timestamps, making it easy to track changes for compliance and debugging.
